About the Application

This application is designed to streamline the testing and management of valves, spec sheets, and test plans. Follow the steps below to make the most of its features.

Getting Started

  1. Create or Identify Your Valve:

    Start by ensuring the valve you want to test is listed in the application. If not, you can add a new valve using the Add Valve page. Provide a name, type, optional API endpoint, and specifications for the valve.

  2. Upload Spec Sheets:

    Spec sheets can be uploaded via the Upload Spec Sheet page. This helps associate critical documentation with your valves.

    • Accepted formats: PDF, DOCX, XLSX
    • Optional: Link the spec sheet to a specific valve by providing its ID.
  3. Upload Test Plans:

    Navigate to the Upload Test Plan page to upload your test plans. These plans will be parsed and stored for execution.

    • Accepted formats: PDF, CSV, TXT
    • Each test plan should include steps like "Open Valve 50%".
  4. Run Test Plans:

    Once your valves and test plans are ready, go to the Run Test Plan page to execute a specific test plan. Provide the ID of the test plan to initiate testing.

  5. View Test Results:

    After running a test plan, you can view the results on the View Results page. Logs and statuses for each step will be displayed.

Additional Features

  • Real-Time Logs: While running test plans, view live logs under the "Real-Time Logs" section on the dashboard.
  • Dashboard Overview: The main Dashboard provides quick access to key features and live updates.
  • Search & Filter: Use the search functionality (if enabled) to quickly locate specific valves, spec sheets, or test plans.

Frequently Asked Questions

What file types are supported for uploads?

Supported file types include:

  • Spec Sheets: PDF, DOCX, XLSX
  • Test Plans: PDF, CSV, TXT
How do I find a valve's ID?

You can view all available valves and their IDs on the View Valves page.

What if a test plan fails?

Check the logs for details about the failure. Ensure the valve and test plan are configured correctly, and try again.
